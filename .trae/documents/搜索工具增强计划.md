# 搜索工具增强计划

## 需求概述

用户希望改进搜索工具的工作流程：

1. `web_search` 工具只返回 URL 和标题（不返回内容）
2. 新增 `fetch_page_with_content` 工具，AI 可以调用它来获取指定 URL 的正文内容（智能提取，去除导航/广告等噪音）
3. AI 的工作流程：先搜索 → 获取 URL 列表 → 自行决定打开前 10 个页面获取内容 → 放到 Agent 提示词中

## 实现步骤

### 步骤 1：修改 `web_search` 工具

- 文件：`g:\ai\packages\server\src\services\tool.service.ts`

- 修改 `searchTool` 的返回格式，只返回 `{ query, provider, results: [{ title, url }] }`（移除 `snippet` 字段）

- 更新工具描述，说明返回的是 URL 列表

### 步骤 2：新增 `fetch_page_with_content` 工具

- 文件：`g:\ai\packages\server\src\services\tool.service.ts`

- 创建新工具 `fetch_page_with_content`：

  - 参数：`url` (string, required) - 要抓取的页面 URL

  - 功能：

    1. 使用 `http.fetch` 获取页面 HTML
    2. 使用智能提取算法（Readability 风格）提取正文内容：

       - 移除 `<script>`, `<style>`, `<nav>`, `<header>`, `<footer>`, `<aside>`, `<form>` 等噪音元素

       - 保留 `<article>`, `<main>`, `<p>`, `<h1>-<h6>`, `<ul>`, `<ol>`, `<li>`, `<blockquote>` 等内容元素

       - 提取纯文本，去除 HTML 标签

    3. 限制返回长度（从 settings 读取，默认 3000 字符）

  - 返回格式：`{ url, title: string, content: string, charCount: number }`

  - 注册到 registry

### 步骤 3：添加内容长度设置项

- 文件：`g:\ai\packages\core\src\models\setting.ts`

- 添加设置项：`'webSearch.maxContentLength': z.number().int().min(500).max(10000).default(3000)`

- 文件：`g:\ai\packages\state\src\index.ts`（如果需要）

### 步骤 4：更新 WebSearchSettings UI

- 文件：`g:\ai\packages\app-ui\src\features\settings\WebSearchSettings.tsx`

- 在提示信息区域添加「内容长度限制」设置项（数字输入框，500-10000 字符）

- 保存到 settings

### 步骤 5：更新 ToolService 读取设置

- 文件：`g:\ai\packages\server\src\services\tool.service.ts`

- 在 `fetch_page_with_content` 工具中从 settings 读取 `webSearch.maxContentLength`

- 用于限制返回内容长度

### 步骤 6：验证编译

- 运行 `pnpm --filter @xiabao/core build`

- 运行 `pnpm --filter @xiabao/server build`

- 运行 `pnpm --filter @xiabao/app-ui typecheck`

## 最终效果

- `web_search`：返回 `{ query, provider, results: [{ title, url }] }`（最多 10 条）

- `fetch_page_with_content`：返回 `{ url, title, content, charCount }`（智能提取正文，限制长度）

- AI 工作流程：搜索 → 选择前 10 个 URL → 调用 `fetch_page_with_content` 逐个获取内容 → 综合分析
